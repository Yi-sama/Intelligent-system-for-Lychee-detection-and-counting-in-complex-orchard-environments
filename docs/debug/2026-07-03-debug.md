## Lychee Training Debug

- Symptom:
  - early double-click runs of `train.bat` disappeared immediately;
  - historical YOLO runs failed with CUDA/memory instability under larger settings;
  - later runs failed with `MemoryError` in the DataLoader or `cv2.error: ... Insufficient memory` while loading images;
  - one verification attempt also showed `conda run` failing with encoding/plugin output issues.
- Investigation summary:
  - old training logs were still present in `runs\lychee`, so some pasted output such as `31/80` came from earlier runs rather than the newest script revision;
  - `lychee2` is detect-format data, while `lychee3` contains segmentation-style labels;
  - mixed detect/segment training is acceptable for the current detect goal because Ultralytics falls back to boxes, but it increases instability and ambiguity during debugging;
  - the machine-side failures were primarily on the Windows host-memory / virtual-memory path, not just the 8 GB GPU VRAM path.
- Root cause:
  - unstable historical settings such as larger `imgsz`, larger `batch`, worker multiprocessing, and default mosaic behavior were too aggressive for the current machine and dataset mix;
  - `conda run` added a separate reliability problem in this Chinese-path environment, so it was not a good launcher for overnight training.
- Resolution:
  - rewrote `train.bat` to use direct `yolo.exe` launch from `D:\Miniconda3\envs\yolo\Scripts\yolo.exe`;
  - kept a conservative training profile in `train.bat` with `epochs=80`, `imgsz=448`, `batch=1`, `workers=0`, `amp=False`, `augment=False`, `mosaic=0.0`, `mixup=0.0`, `copy_paste=0.0`, `close_mosaic=0`;
  - preserved merged training through `lychee dataset\combined-detect.yaml` because the user wants both datasets used.
- Durable findings to promote:
  - do not rely on `conda run` for this training launcher on the current machine;
  - verify `runs\lychee\yolo26_merged_detect_safe\args.yaml` before trusting copied console output from older runs;
  - if overnight training still fails, the next likely mitigation is machine memory/pagefile relief or simplifying the dataset mix before changing the GUI.
